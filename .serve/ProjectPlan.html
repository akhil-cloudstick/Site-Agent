▶Build waves — dependency-ordered (Wave 0 = no prerequisites; each later wave needs the earlier ones)

Topological order from the DEPS graph. A task in Wave k has its longest prerequisite chain k deep — it can't truly start until everything in earlier waves it depends on is done. Within a wave, ordered by priority. Click an id to jump to the task.

Self-contained & offline — no external requests. Progress = done leaf-tasks ÷ all non-deferred leaf-tasks; in-progress and blocked count as not done. State is per-browser (localStorage); use Export to back it up. This is a planning artifact only — it generates no product code.
